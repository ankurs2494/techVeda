Content Delivery Network (CDN): Beginner to SRE-Level Complete Guide

A complete CDN guide from basics to advanced SRE concepts with diagrams, use cases, examples, and 20 interview Q&A.
January 4, 2026 · Ankur Singh

Introduction#

A Content Delivery Network (CDN) is a core building block of modern, scalable, and reliable web systems. From simple websites to global-scale platforms like Netflix and Cloudflare, CDNs are essential for performance, availability, and security.

This guide starts from absolute basics and progresses to advanced SRE-level concepts, including cache strategies, failure handling, and real-world operations.

What is a CDN? (Beginner Level)#

A CDN is a globally distributed network of servers that deliver content to users from the nearest geographical location.

Instead of every user hitting your origin server, content is served from edge locations, reducing latency and load.

Simple Example#

  • User in India requests image.png
  • Image is served from Mumbai edge instead of US origin

What Does a CDN Cache?#

  • Static assets (JS, CSS, images, videos)
  • HTML pages
  • API responses (with rules)
  • Media streams

Key Benefits of CDN#

Performance#

  • Reduced latency
  • Faster page load times

Scalability#

  • Handles traffic spikes
  • Offloads origin servers

Reliability#

  • Origin protection
  • Failover and redundancy

Security#

  • DDoS protection
  • WAF integration
  • TLS termination

CDN Use Cases#

Beginner / Common#

  • Website static asset delivery
  • Image & video hosting

Intermediate#

  • API acceleration
  • Mobile app backend caching

Advanced / SRE-Level#

  • Global SaaS platforms
  • Live streaming
  • Zero-downtime deployments
  • Edge computing

Real-Life CDN Examples#

  • Netflix: Video streaming from edge locations
  • Amazon CloudFront: E-commerce asset delivery
  • Cloudflare: CDN + security platform
  • YouTube: Adaptive video delivery

CDN Caching Strategies (Intermediate)#

  • Cache-Control headers
  • TTL-based caching
  • Cache busting (versioned URLs)
  • Purge / Invalidation

Example Header#

Cache-Control: public, max-age=3600

Advanced CDN Concepts (SRE Level)#

Edge Computing#

  • Run logic at CDN edge (Cloudflare Workers, Fastly Compute)

Origin Shield#

  • Single mid-tier cache to protect origin

Cache Hierarchy#

  • Edge → Regional → Origin

Failover & Multi-CDN#

  • Traffic switching on failure
  • Vendor lock-in avoidance

Common CDN Pitfalls#

  • Incorrect cache headers
  • Serving stale content
  • Cache poisoning
  • Over-aggressive caching

Interview Questions & Answers (20 Must-Know)#

1. What is a CDN? A CDN is a distributed network that serves content from locations closest to users.
2. Why do we use CDNs? To reduce latency, improve availability, and protect origin servers.
3. What is an edge location? A CDN server that delivers content close to users.
4. What is an origin server? The main backend server where original content resides.
5. What is cache hit and miss? Hit means content served from cache; miss means fetched from origin.
6. What headers control CDN caching? Cache-Control, Expires, ETag, Last-Modified.
7. What is cache invalidation? Removing cached content before TTL expires.
8. CDN vs Load Balancer? CDN serves cached content; load balancer distributes traffic to backends.
9. Can CDNs cache dynamic content? Yes, with rules, headers, and edge logic.
10. What is edge computing? Executing code closer to users at CDN edge locations.
11. What is Origin Shield? A centralized cache layer protecting origin servers.
12. How do CDNs help with DDoS? By absorbing and filtering traffic at the edge.
13. What is multi-CDN? Using multiple CDN providers for resilience and performance.
14. What is cache poisoning? Injecting malicious content into CDN cache.
15. What is stale-while-revalidate? Serving stale content while fetching fresh data in background.
16. How does CDN failover work? Traffic is rerouted to healthy edges or another CDN.
17. What metrics do SREs monitor for CDN? Cache hit ratio, latency, error rate, origin load.
18. CDN vs Reverse Proxy? CDN is globally distributed; reverse proxy is usually local.
19. What happens if CDN is down? Traffic can fall back to origin or another CDN.
20. One-line CDN summary? CDNs bring content closer to users for speed, scale, and security.

Conclusion#

For SREs, CDNs are not just about speed—they are critical infrastructure for reliability, scalability, and security. Mastering CDN internals is essential for operating global systems.
